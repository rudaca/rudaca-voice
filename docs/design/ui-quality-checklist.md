# Rudaca Voice UI Quality Checklist

## General
- Screen has clear title and short helper text.
- Main call-to-action is obvious.
- Page width feels intentional.
- Spacing is consistent.
- No starter-kit placeholder UI remains.
- Empty states are polished and helpful.

## Idea Cards
- Idea title is easy to scan.
- Status badge is clear and consistent.
- Vote count is visible but not oversized.
- Metadata line is readable:
  Board Group · Board · Category · Comments · Date
- Cards have consistent padding, border, radius, and hover state.
- Anonymous author display is clean.

## Forms
- Form sections are grouped logically.
- Labels and helper text are clear.
- Validation errors are easy to understand.
- Submit/cancel buttons are placed consistently.

## Manager Screens
- Review actions are visually separated from public idea content.
- Status/priority/impact/effort controls are easy to scan.
- History is readable but not visually dominant.

## Responsive
- The screen works well on desktop first.
- The layout does not break on tablet or narrow browser widths.
- On smaller screens, cards stack cleanly.
- Filters wrap or collapse cleanly.
- Buttons do not overflow.
- Metadata lines do not become messy.
- Forms remain usable on mobile.
- Admin-heavy screens can be optimized for desktop, but should not be broken on mobile.
